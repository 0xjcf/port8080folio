Themeable Syntax Highlighter

CSS Custom Properties Support:
The syntax highlighter now uses CSS custom properties (CSS variables) for all colors.
You can customize any theme by overriding these variables in your CSS.
Below are examples showing different theme implementations.

Default Theme

const coffeeShopMachine = setup({ types: {} as { context: { orders: Order[]; // Pink (#E06C75) inventory: Record; // Pink (#E06C75) revenue: number; // Pink (#E06C75) }; events: | { type: 'PLACE_ORDER'; items: string[] } // Blue (#79B8FF) | { type: 'CANCEL_ORDER'; orderId: string } // Blue (#79B8FF) | { type: 'PAYMENT_RECEIVED'; amount: number }; // Blue (#79B8FF) }, actions: { processOrder: assign({ orders: ({ context, event }) => [ ...context.orders, // Pink { id: Date.now(), items: event.items, status: 'pending' } // Blue ], inventory: ({ context, event }) => { const updated = { ...context.inventory }; // Pink event.items.forEach(item => { // Blue updated[item] = context.inventory[item] - 1; // Pink }); return updated; } }), logActivity: enqueueActions(({ enqueue, event }) => { enqueue('saveToDatabase'); // Yellow (#F0DB4F) enqueue('updateAnalytics'); // Yellow (#F0DB4F) }) } });

Available CSS Variables

/* XState specific */
--xstate-keyword      /* context, event, spawn, etc */
--context-property    /* properties in context */
--event-property      /* properties in event */
--state-name          /* state names */
--event-name          /* EVENT_NAMES */
--service-name        /* services */

/* JavaScript/TypeScript */
--keyword             /* const, let, if, etc */
--string              /* strings */
--number              /* numbers */
--comment             /* comments */
--function            /* function calls */
--operator            /* operators */
--punctuation         /* brackets, commas */
--variable            /* variables */
--property            /* object properties */
--boolean             /* true/false */
--null                /* null/undefined */
--default             /* default text */
